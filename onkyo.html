Onkyo Controller – IoT System Case Study

A real-time mobile application designed to control IoT-based AV receivers over a local network. Focused on low-latency communication, high reliability, and seamless device interaction.

Flutter IoT System Design Clean Architecture Real-time Communication

Problem Statement

  • Devices operate on unstable local networks
  • High latency affects user experience
  • No consistent state between device and UI
  • Frequent disconnections and packet loss

Solution Overview

  • Designed real-time communication layer using TCP/UDP
  • Implemented state synchronization engine
  • Built offline-first interaction handling
  • Ensured low latency command execution

Architecture

Clean Architecture Layers:
  • Presentation: Flutter UI + State Management
  • Domain: Business logic, use cases
  • Data: Repository + network/device communication
Patterns Used:
  • Repository Pattern
  • Dependency Injection
  • Modular Structure

System Design

  • Real-time device communication over local network
  • Command queue system for reliability
  • State sync between device & UI
  • Retry mechanism for failed commands

Key Engineering Challenges

  • Handling unstable WiFi connections
  • Maintaining consistent device state
  • Low latency execution (<100ms)
  • Efficient parsing of device responses

Solutions Implemented

  • Reconnection strategy with exponential backoff
  • Local caching of device state
  • Optimized network calls
  • Error handling for device unavailability

Impact

  • Smooth real-time device control
  • Reduced latency significantly
  • Improved reliability of communication
  • Production-ready scalable architecture

Key Learnings

  • Designing for unreliable networks
  • Building real-time systems in mobile
  • Importance of architecture for scalability
